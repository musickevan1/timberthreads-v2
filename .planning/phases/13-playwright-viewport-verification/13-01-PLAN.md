  <action>
Create `tests/viewport.spec.ts` with assertion-only tests. NO screenshot comparisons — only element visibility and content assertions.

The test file should contain these test groups:

**1. Desktop tests (run in `desktop-1280` project via `test.describe`):**

Use `test.describe('desktop viewport', () => { ... })` and filter by project name within each test or use a top-level describe for clarity. Since Playwright runs all tests in all projects by default, use `test.skip` based on `testInfo.project.name` to restrict viewport-specific tests, OR better: keep tests generic where possible and only skip viewport-specific ones.

Recommended approach: Write tests that work at any viewport where the feature is present. Use `testInfo.project.name` to gate viewport-specific checks.

```typescript
import { test, expect } from '@playwright/test';

test.beforeEach(async ({ page }) => {
  await page.goto('/');
});
```

**Test: Header brand text visible**
- Assert `page.locator('header').getByText('Timber & Threads')` is visible.
- This test should pass at ALL viewports (brand text always visible per Phase 9 fix).

**Test: Desktop navigation links visible (desktop only)**
- Skip if project name is not `desktop-1280`.
- Assert `page.locator('header nav ul')` is visible (desktop nav links are `hidden md:flex` — visible at 1280px, hidden at 375px/320px).

**Test: Pricing calculator section visible**
- Assert `page.locator('#pricing-calculator')` is visible after scrolling to it.
- Works at all viewports.

**Test: Get a Quote button visible**
- Assert `page.getByRole('button', { name: 'Get a Quote' })` is visible after scrolling calculator into view.
- Works at all viewports (calculator has default valid inputs: 4 guests, 2 nights).

**Test: Map iframe loads after scroll**
- Scroll `page.locator('#maps-iframe')` into view using `scrollIntoViewIfNeeded()`.
- Wait briefly for IntersectionObserver to fire.
- Assert the iframe element is visible: `await expect(page.locator('#maps-iframe')).toBeVisible()`.
- Works at all viewports.

**Test: Contact form visible**
- Scroll `page.locator('#contact-form')` into view.
- Assert it is visible.
- Works at all viewports.

**Test: No horizontal overflow (mobile only)**
- Skip if project name is `desktop-1280`.
- Assert `document.body.scrollWidth <= document.documentElement.clientWidth` via `page.evaluate()`.
- This satisfies MOBL-02 (375px) and MOBL-03 (320px).

**Test: Get a Quote pre-fills contact message**
- Wait for `page.getByRole('button', { name: 'Get a Quote' })` to be visible (Preact hydration).
- Click the button.
- Wait for `page.locator('#message')` to not have value `''` (auto-retry covers smooth scroll + 600ms timer).
- Read `inputValue()` from `#message` textarea.
- Assert it contains `'Group Size:'` and `'Estimated Total:'`.
- This test should run at all viewports — validates the cross-island communication works everywhere.

**Important implementation details:**
- Do NOT use `page.waitForTimeout()` — rely on Playwright's auto-retry in `toBeVisible()` and `not.toHaveValue('')`.
- Scope header brand text selector to `page.locator('header')` to avoid matching body text occurrences (Pitfall 5 from research).
- Use `scrollIntoViewIfNeeded()` before asserting lazy-loaded elements (map iframe, contact form below fold).
- The calculator renders with default values (4 guests, 2 nights, no meals) — the "Get a Quote" button is visible with these defaults because they meet minimums.
- For the pre-fill assertion, use `await expect(message).not.toHaveValue('')` (auto-retry) BEFORE reading `inputValue()`, since the smooth scroll and focus timer add ~600ms delay.
