**Step 3: Create src/pages/api/contact.ts**

This is the server endpoint. Key implementation details:

```typescript
import type { APIRoute } from 'astro';
import { Resend } from 'resend';

export const prerender = false;  // CRITICAL: opts this route out of static generation
```

The `POST` export handles the request with this flow:

1. **Parse request body** -- wrap `request.json()` in try/catch. On parse failure, return 400 `{ message: 'Invalid request body' }`.

2. **Honeypot check** -- if `body.website` has any value (bots fill this, humans never see it), return **200** with `{ message: 'Message sent successfully!' }`. Return 200 (not 400) so bots think their submission worked (per research recommendation). Do NOT send any email.

3. **Server-side validation:**
   - `name`: must be a non-empty trimmed string
   - `email`: must be non-empty and match `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
   - `message`: must be a non-empty trimmed string
   - Collect all errors into an array. If any, return 400 with `{ message: errors.join('. ') }`.

4. **Send email via Resend:**
   - Instantiate: `const resend = new Resend(import.meta.env.RESEND_API_KEY);`
   - Call `resend.emails.send()` with:
     - `from`: `'Timber & Threads Contact <noreply@timberandthreadsretreat.com>'`
     - `to`: `[import.meta.env.OWNER_EMAIL]`
     - `replyTo`: the visitor's email (so owner can reply directly)
     - `subject`: `New inquiry from ${name}`
     - `html`: formatted HTML with name, email (as mailto link), and message (newlines converted to `<br>`)
     - `text`: plain text fallback
   - Check the `{ data, error }` return. If `error`, log it with `console.error('Resend error:', error)` and return 500 with `{ message: 'Failed to send your message. Please try again or call us directly.' }`.

5. **Success response:** Return 200 with `{ message: 'Message sent successfully!' }`.

All responses must include `headers: { 'Content-Type': 'application/json' }`.

**CRITICAL: Do NOT modify astro.config.mjs.** The CONTEXT.md says to switch to `output: 'hybrid'` but research confirmed this is INVALID in Astro 5. Keep `output: 'static'` as-is. The `export const prerender = false` in the endpoint file is what makes it serverless. Using `output: 'hybrid'` per user decision (research found `output: 'hybrid'` was removed in Astro 5 -- `output: 'static'` with per-route `prerender = false` is the Astro 5 replacement).

**CRITICAL: Use `import.meta.env.RESEND_API_KEY`** (not `process.env`) -- Astro server endpoints use `import.meta.env` for env var access. Never use `PUBLIC_` prefix for the API key (CNTC-06).

**HTML email sanitization:** Escape `<`, `>`, `&` in user-provided name and message before inserting into the HTML email body to prevent HTML injection. Use a simple replace chain: `str.replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;')`. The text version needs no escaping.
