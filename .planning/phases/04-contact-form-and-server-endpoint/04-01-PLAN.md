**Add `<script>` block to Contact.astro with client-side validation and form submission:**

The script block goes at the bottom of Contact.astro. It handles:

1. **On-blur validation** (per research recommendation for older demographic):
   - Name field blur: if empty, show "Please enter your name" in `#name-error`
   - Email field blur: if empty or fails regex `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`, show "Please enter a valid email address" in `#email-error`
   - Message field blur: if empty, show "Please enter a message" in `#message-error`
   - When field has valid content on blur, hide the error span
   - Do NOT validate on keypress/input -- only on blur (per research: avoids interrupting slow typists)

2. **On-submit validation** (re-validates all fields):
   - Prevent default form submission
   - Run all three field validations
   - If any field invalid, focus the first invalid field and stop
   - If all valid, proceed to fetch

3. **Form submission via fetch:**
   - Collect form data including honeypot `website` field
   - POST to `/api/contact` with JSON body `{ name, email, message, website }`
   - Set `Content-Type: application/json`
   - Before fetch: disable submit button, change text to "Sending...", show spinner, hide error banner
   - On success (res.ok): hide the form element, show `#success-message` with inline confirmation (per research -- inline replacement, NOT toast)
   - On error (res not ok): parse JSON for error message, show in `#error-message` banner above form, DO NOT reset form inputs (per user decision)
   - On network error (catch): show "Network error. Please check your connection and try again." in error banner
   - Finally block: re-enable button, restore "Send Message" text, hide spinner

4. **TypeScript casting:** Use `as HTMLFormElement`, `as HTMLButtonElement`, etc. for DOM queries (matches existing Gallery.astro pattern)

**Important:** The form will NOT actually work end-to-end yet (no server endpoint). The fetch will 404. This is expected -- Plan 02 creates the endpoint. The UI, validation, spinner, and error handling are all testable independently.
