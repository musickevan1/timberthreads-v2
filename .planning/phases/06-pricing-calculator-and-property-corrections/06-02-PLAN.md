**Step 2: Create src/components/PricingCalculator.tsx**

Build a Preact functional component with the following structure:

```tsx
/** @jsxImportSource preact */
import { useState } from 'preact/hooks';
```

**State:**
- `groupSize: number` — default 6 (middle of 4-12 range)
- `nights: number` — default 3 (middle of 2-7 range)
- `includeMeals: boolean` — default false

**Pricing logic:**
```
if (groupSize <= 10) {
  nightlyRate = includeMeals ? 75 : 60;
  totalNightly = nightlyRate * groupSize;
  foodCost = includeMeals ? 15 * groupSize * nights : 0;
  accommodationCost = 60 * groupSize * nights;
  total = accommodationCost + foodCost;
} else {
  // Groups of 10-12: $600/night flat rate
  totalNightly = 600;
  accommodationCost = 600 * nights;
  foodCost = includeMeals ? 15 * groupSize * nights : 0;
  total = accommodationCost + foodCost;
}
```

**UI structure (use Tailwind classes matching site design):**

1. **Inputs panel** — left/top on mobile:
   - **Group Size slider**: `<input type="range" min={4} max={12} step={1}>` with current value displayed. Label: "Group Size" with "Min 4" hint near the low end.
   - **Nights slider**: `<input type="range" min={2} max={7} step={1}>` with current value displayed. Label: "Number of Nights" with "Min 2" hint near the low end.
   - **Meals toggle**: A styled toggle switch (not checkbox, not radio). Label: "Include meals (+$15/night per person)". Use a `<button>` with role="switch" and aria-checked for accessibility.

2. **Breakdown panel** — right/bottom on mobile:
   - Card/panel with white background, rounded corners, shadow
   - Line items:
     - For ≤10: "Accommodation: {groupSize} guests × ${nightlyRate}/night × {nights} nights = ${accommodationCost}"
     - For 10-12: "Accommodation: $600/night × {nights} nights = ${accommodationCost}" with note "Groups of 10–12: flat rate of $600/night"
     - If meals toggled: "Meals: {groupSize} guests × $15/night × {nights} nights = ${foodCost}"
     - Divider line
     - "Estimated Total: ${total}" in larger/bold text
   - When group size crosses from ≤10 to 11+, show a brief note: "Groups of 10–12: flat rate of $600/night" — this should appear smoothly, not jarring.
   - Disclaimer at bottom of breakdown: "This is an estimate. Final pricing confirmed upon booking." in small muted text.

3. **Styling guidelines:**
   - Use `font-serif` for headings (matches site)
   - Use `text-stone-*` color palette (matches site)
   - Use `text-brand` / `bg-brand` for accent colors (matches site)
   - Slider track should be styled with brand color for the filled portion (use CSS pseudo-elements or inline style for the gradient trick: `background: linear-gradient(to right, var(--color-brand) {percent}%, #d6d3d1 {percent}%)`)
   - Toggle switch: pill shape, brand color when on, stone-300 when off
   - All values update in real-time as sliders move (onChange handler)
   - Layout: `grid md:grid-cols-2 gap-8` for inputs/breakdown side by side on desktop, stacked on mobile

4. **Accessibility:**
   - Slider inputs have `aria-label` and `aria-valuemin`/`aria-valuemax`/`aria-valuenow`
   - Toggle has `role="switch"` and `aria-checked`
   - Breakdown values in an `aria-live="polite"` region so screen readers announce changes
   - All text meets contrast requirements (stone-600+ on white)

**Do NOT:**
- Use any external UI library (no Headless UI, no Radix) — plain Preact + Tailwind
- Add server-side logic — this is a pure client-side calculator
- Import React — use Preact (lighter, same API for this use case)
