# Plan 08-03: SEO Markup, sitemap.xml, and Production Deployment

## Goal

Complete the SEO layer (enhanced meta tags, LocalBusiness JSON-LD structured data, robots.txt, sitemap.xml, Open Graph tags with hero OG image), verify semantic HTML heading hierarchy is complete, then deploy to Vercel preview for client approval, followed by DNS cutover to timberandthreadsretreat.com.

## Context

**What already exists:**
- `astro.config.mjs` has `output: 'static'` and Vercel adapter — Vercel deploys are already working
- `BaseLayout.astro` has basic OG tags (og:type, og:url, og:title, og:description, og:image) referencing `/og-default.jpg` — but `og-default.jpg` doesn't exist in `public/` yet
- `public/robots.txt` exists with minimal content (`User-agent: * / Allow: /`) — needs sitemap reference added
- `index.astro` uses default title and description from BaseLayout — needs updated title format and better meta description
- No `sitemap.xml` yet — Astro has `@astrojs/sitemap` integration available
- No JSON-LD structured data yet

**Go-live strategy (from CONTEXT.md, locked decisions):**
1. Build and deploy to Vercel preview URL (this happens automatically on every push — CI/CD is already configured)
2. Human verifies preview URL
3. Human adds domain in Vercel dashboard
4. Human updates Cloudflare DNS to point to Vercel
5. Site goes live at timberandthreadsretreat.com

**The OG image:** Use the hero image as the OG image. The hero image (`src/assets/images/hero-image.png`) is processed by Astro and output to `dist/_astro/`. For OG images, we need a publicly accessible URL. The simplest approach: copy the hero image to `public/og-image.jpg` as a static file (not processed by Astro's image pipeline) so it's accessible at the `/og-image.jpg` URL. OG images should be 1200x630px recommended, JPG or PNG. If hero-image.png is close to this aspect ratio, use it directly.

**Sitemap:** Install `@astrojs/sitemap`, add to integrations, set `site` in `astro.config.mjs` — Astro will auto-generate `sitemap-index.xml` and `sitemap-0.xml` on build.

## Discovery Level

Level 1 — `@astrojs/sitemap` is a known Astro first-party package with straightforward integration. Quick verification only.

**First, add `canonicalURL` to the frontmatter of `BaseLayout.astro`.**

The frontmatter already has prop declarations. Add the canonical URL computation immediately after the props destructuring:

```astro
---
// existing props destructuring, e.g.:
const { title, description, ogImage = '/og-image.jpg' } = Astro.props;

// Derive canonical URL from the current page path + configured site URL
const canonicalURL = new URL(Astro.url.pathname, Astro.site).toString();
---
```

`Astro.site` is the `site` value set in `astro.config.mjs` (`https://timberandthreadsretreat.com`). `Astro.url.pathname` is the current page path (e.g., `/`). This produces `https://timberandthreadsretreat.com/` for the home page.

Replace the Open Graph block and add JSON-LD in `BaseLayout.astro`:

```html
<!-- Canonical URL -->
<link rel="canonical" href={canonicalURL} />

<!-- Open Graph -->
<meta property="og:type" content="website" />
<meta property="og:url" content={canonicalURL} />
<meta property="og:title" content={title} />
<meta property="og:description" content={description} />
<meta property="og:image" content={`https://timberandthreadsretreat.com${ogImage}`} />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
<meta property="og:site_name" content="Timber & Threads Retreat" />
<meta property="og:locale" content="en_US" />

<!-- Twitter Card -->
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content={title} />
<meta name="twitter:description" content={description} />
<meta name="twitter:image" content={`https://timberandthreadsretreat.com${ogImage}`} />
```
