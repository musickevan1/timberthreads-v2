# Plan 08-02: Accessibility Audit and Fixes (WCAG AA)

## Goal

Achieve WCAG AA compliance across the entire site: 44x44px touch targets on all interactive elements, passing color contrast ratios, click-to-call phone links, keyboard navigation for all interactive elements including the pricing calculator, and descriptive alt text on every image.

## Context

The site already has solid accessibility foundations from v2.0:
- `aria-label`, `aria-expanded`, `role` attributes on nav/hamburger
- `aria-hidden="true"` on decorative SVG icons
- Form field labels with `for` associations
- `aria-live="polite"` on the calculator's output panel
- `role="alert"` on error messages

Known gaps to address in this plan:
- **Touch targets (A11Y-02):** The hamburger button is `w-10 h-10` (40x40px) — needs to be 44x44px. Desktop nav links have no explicit minimum height. CTA links need padding verified.
- **Color contrast (A11Y-03):** Brand color `#0D9488` on white background — need to verify contrast ratio. Stone-500 and Stone-600 text on white needs checking. Footer link text (stone-400) is likely failing.
- **Click-to-call (A11Y-05):** The phone number in Contact.astro already has `href="tel:+14173431473"` — confirm it renders as a proper link. Verify it's not hidden on mobile.
- **Keyboard navigation (A11Y-06):** PricingCalculator sliders use native `<input type="range">` — keyboard navigable by default. The meals toggle is a `<button role="switch">` — keyboard navigable. The mobile nav uses `focus()` trap on Escape key. Verify the lightbox (PhotoSwipe) has keyboard support (it does by default). Add `:focus-visible` styles for non-mouse users.
- **Alt text (A11Y-04):** Check every `<Image>` component and `<img>` tag across all components — verify all have meaningful, non-empty alt attributes.

## Discovery Level

Level 0 — no new dependencies. All WCAG AA requirements are pure CSS and HTML attribute fixes.

**WCAG AA Contrast Requirements:**
- Normal text (< 18pt / < 14pt bold): 4.5:1 minimum ratio
- Large text (>= 18pt / >= 14pt bold): 3:1 minimum ratio

**Fix 1: Focus-visible styles (A11Y-06)**

Add global focus-visible styles to `src/styles/global.css`. Place after the heading defaults block:

```css
/* Focus indicators: visible for keyboard users, hidden for mouse clicks */
:focus-visible {
  outline: 3px solid var(--color-brand);
  outline-offset: 3px;
  border-radius: 3px;
}

/* Remove default focus outline on non-keyboard focus */
:focus:not(:focus-visible) {
  outline: none;
}
```

This ensures keyboard users always see a clear focus ring (brand teal), while mouse clicks don't show outlines.
