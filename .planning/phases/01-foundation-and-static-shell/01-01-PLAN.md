  <action>
    1. Scaffold the Astro project from the project root `/home/evan/Projects/clients/timberandthreads-v2/`:
       ```
       npm create astro@latest -- . --template minimal --no-install --no-git
       ```
       Use `--template minimal` for a clean starting point. The `.` targets the current directory.

    2. Install dependencies and add integrations:
       ```
       npm install
       npx astro add tailwind --yes
       npx astro add vercel --yes
       ```
       `npx astro add tailwind` on Astro 5.2+ installs `@tailwindcss/vite` (NOT the deprecated `@astrojs/tailwind`). Verify `package.json` does NOT contain `@astrojs/tailwind`.

    3. Also install the self-hosted Inter font package:
       ```
       npm install @fontsource-variable/inter
       ```

    4. Configure `astro.config.mjs` (the `npx astro add` commands may have already partially configured this -- merge carefully):
       ```javascript
       import { defineConfig } from 'astro/config';
       import tailwindcss from '@tailwindcss/vite';
       import vercel from '@astrojs/vercel';

       export default defineConfig({
         output: 'static',
         adapter: vercel(),
         vite: {
           plugins: [tailwindcss()],
         },
         image: {
           domains: ['res.cloudinary.com'],
         },
       });
       ```
       CRITICAL: `output: 'static'` MUST be set. Without it, Vercel treats the site as serverless.
       CRITICAL: Import vercel from `'@astrojs/vercel'` (NOT `'@astrojs/vercel/static'` -- the subpath was removed).

    5. Verify `tsconfig.json` has `"strict": true` or extends an Astro preset that enables strict. Astro 5 defaults to strict mode, so this should already be correct. If the template generates `"extends": "astro/tsconfigs/strict"`, that is correct.

    6. Create `public/favicon.svg` -- a simple quilting-themed SVG placeholder:
       ```svg
       <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
         <rect width="32" height="32" rx="4" fill="#0D9488"/>
         <text x="16" y="22" text-anchor="middle" font-size="18" font-family="serif" fill="white">T</text>
       </svg>
       ```

    7. Create `public/robots.txt`:
       ```
       User-agent: *
       Allow: /
       ```

    8. Update `.gitignore` to include Astro/Vercel patterns (the scaffold may already generate one -- ensure it includes `node_modules/`, `dist/`, `.astro/`, `.vercel/`).

    9. Verify the scaffold works:
       ```
       npm run build
       ```
       This must complete with zero errors. The build output should be in `dist/`.

<task type="auto">
  <name>Task 2: Configure Tailwind v4 global CSS with brand theme</name>
  <files>

  <action>
    Create `src/styles/global.css` with the Tailwind v4 CSS-first configuration. This is the ONLY place Tailwind is configured -- no `tailwind.config.js` file should exist.

    ```css
    @import "tailwindcss";

    @theme {
      /* Brand color: #0D9488 is the exact hex from the current site.
         Tailwind v4 changed teal-600 to oklch(~#009689), so we define
         the brand color explicitly to preserve visual identity. */
      --color-brand: #0D9488;
      --color-brand-dark: #0B7D74;
      --color-brand-light: #14B8A6;

      /* Typography */
      --font-sans: 'Inter Variable', ui-sans-serif, system-ui, sans-serif;
      --font-serif: Georgia, 'Times New Roman', serif;
    }

    /* Base styles */
    html {
      scroll-behavior: smooth;
    }

    body {
      font-family: var(--font-sans);
      font-size: 1rem;
      line-height: 1.6;
      color: var(--color-stone-800);
      background-color: white;
    }

    /* Heading defaults */
    h1, h2, h3, h4, h5, h6 {
      font-family: var(--font-serif);
    }

    h1 {
      font-weight: 500;
    }

    h2, h3, h4, h5, h6 {
      font-weight: 400;
    }
    ```

    IMPORTANT: Do NOT create a `tailwind.config.js` file. Tailwind v4 uses CSS-first configuration via `@theme`. If `npx astro add tailwind` created one, delete it.

    Verify that the `@import "tailwindcss"` and `@theme` directives are recognized by running `npm run dev` and confirming no CSS errors in the terminal.
  </action>

  </verify>

  </done>
