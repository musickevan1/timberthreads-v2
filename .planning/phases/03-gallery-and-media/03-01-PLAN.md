  <name>Task 1: Create gallery data file with Cloudinary URL helpers and typed image array</name>
  <files>src/data/gallery.ts</files>
  <action>
Create `src/data/gallery.ts` with three sections:

**1. Cloudinary URL helper functions:**
- `CLOUD_NAME` constant set to `'timberandthreads'` (placeholder — single place to update when real account is configured)
- `cloudinaryUrl(publicId: string, transforms: string): string` — builds full Cloudinary URL: `https://res.cloudinary.com/${CLOUD_NAME}/image/upload/${transforms}/${publicId}`
- `thumbUrl(publicId: string, size = 400): string` — calls cloudinaryUrl with transforms `c_fill,g_auto,w_${size},h_${size}/f_auto/q_auto` for uniform square crops with AI gravity
- `fullUrl(publicId: string, maxWidth = 2000): string` — calls cloudinaryUrl with transforms `w_${maxWidth}/f_auto/q_auto` for lightbox full-size images
- `srcsetUrl(publicId: string, width: number): string` — calls cloudinaryUrl with `w_${width}/f_auto/q_auto` for responsive srcset width variants

**2. TypeScript types:**
- `GalleryCategory` type: `'facility' | 'quilting'`
- `GalleryImage` interface: `{ id: string, cloudinaryPublicId: string, width: number, height: number, alt: string, caption: string, category: GalleryCategory, order?: number }`

**3. Hardcoded image data array:**
- `galleryImages: GalleryImage[]` with placeholder entries — 4 facility images and 4 quilting images
- Use realistic Cloudinary public IDs like `retreat/facility/front-view`, `retreat/facility/workspace`, `retreat/quilting/quilt-display-1` etc.
- Stub dimensions as `width: 4000, height: 2667` (3:2 landscape typical for DSLR) — these will be replaced with real dimensions when photos are uploaded
- Write meaningful alt text and captions for each placeholder (describe what the photo would show at a retreat)
- `facilityImages` and `quiltingImages` as filtered/sorted exports: `galleryImages.filter(img => img.category === 'facility').sort((a, b) => (a.order ?? 0) - (b.order ?? 0))`

Export all types, helpers, and data arrays.
